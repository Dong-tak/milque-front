### PRD: Relationship-Based Mind Map Interface

---

### 1. **프로젝트 개요**

**프로젝트 이름**: 관계 기반 마인드맵 인터페이스  
**목적**: 사용자들이 노드 간 관계를 시각적으로 파악하고, 마인드맵 구조를 유연하게 확장 및 편집할 수 있는 인터페이스를 제공하는 것을 목표로 합니다. 중심 노드를 기준으로 하위 노드와 외부 노드를 직관적으로 연결하고, 부모-자식 관계와 형제 관계를 효과적으로 표현할 수 있는 시각적 도구를 개발합니다. 사용자가 화살표로 관계를 정리한 것을 마인드맵으로 시각화하여 정리할 수 있게 도와줍니다.

**주요 기능**:

1. 마인드맵을 구성하는 각 노드의 자동 정렬과 배치
2. 부모-자식, 형제 간 관계를 시각적으로 직관적으로 나타냄
3. 외부 노드를 마인드맵 내 특정 위치로 포함할 수 있는 드래그 앤 드롭 기능
4. 화살표로 이어진 객체들의 정보를 기반으로 마인드 맵으로 변환함

---

### 2. **유저 플로우**

1. **마인드맵 진입**:

   - 사용자는 원하는 root(시작) 객체를 선택하면 해당 객체 하위(자식) 관계들을 모두 마인드 맵화 할 수 있습니다.

2. **노드 추가**:

   - 중심 노드에 하위 노드(첫째 자식 노드 포함)를 추가하거나 기존 노드에 형제 노드를 추가합니다.

3. **외부 노드 포함**:

   - 특정 노드를 클릭하여 외부 노드를 드래그 앤 드롭으로 가져오거나, 클릭으로 부모-자식 관계를 맺어 새로운 노드로 추가합니다.

4. **관계 수정**:

   - 노드 간 관계를 변경하거나 순서를 조정하여 마인드맵의 구조를 재편할 수 있습니다.

5. **미리보기 애니메이션**:
   - 노드나 관계를 변경할 때 시각적 피드백을 제공합니다.

---

### 3. **핵심 기능**

#### 1) **노드 인터페이스 및 구조 정의**

- **노드 (Node)**: 각 위젯은 부모 노드와의 포함 관계를 통해 위치가 자동 조정됩니다.

  - **Root**: 중심 노드로, 다른 노드와 달리 이동 가능하며 이외의 노드는 이동 불가합니다.
  - **First 자식**: 첫째 자식 노드는 좌우 구분이 가능하며, 형제 관계와 구분됩니다.

- **DBrelate 관계 관리**:

  - **요소 관계**: 모든 노드 및 보드 간의 관계를 포함한 데이터베이스 구조 관리
  - **포함 관계**: 부모와 자식 간의 계층적 관계 유지
  - **순서 참조 관계**: 형제 노드 간 순서 참조를 통해 노드 배치 순서를 정의

- **가상 박스 (Box)**:
  - **GenBox**: 같은 세대의 형제 노드들을 포함하는 박스
  - **SibBox**: 같은 부모를 공유하는 형제 노드를 포함하는 박스

#### 2) **간격 및 배치 규칙 설정**

- **노드 간 간격**:

  - **GenGap (48px)**: 세대 박스 간 수평 공간
  - **SibGap (24px)**: 형제 노드 간 수직 공간
  - **NavGap (36px)**: 같은 세대 내에서 조상을 공유하는 SibBox 간 수직 공간

- **PositionFix 규칙**:
  - `Root` 노드는 이동 가능하며, 이를 제외한 모든 노드는 고정된 위치를 가집니다.
  - **노드 위치 계산**:
    - `x^n_i, y^n_i`: 세대 및 순서를 기반으로 각 노드의 좌표가 자동으로 계산됩니다.
    - **FamilyBox 크기 계산**:
      - 각 세대별 노드 중 최대 너비와 다음 세대의 FamilyBox 너비의 합으로 구성

#### 3) **유저 액션 및 인터랙션**

- **노드 생성 (CreateNode)**:

  - **형제 노드 추가**: 선택된 노드와 같은 부모를 공유하며 순서를 참조하여 생성
  - **막내 자식 노드 추가**: 마지막 자식 노드를 참조하여 하위 노드로 생성

- **외부 노드 포함**:
  - 외부 노드를 드래그하거나 특정 위치에 추가하여 포함 관계를 형성

#### 4) **미리보기 및 애니메이션**

- **미리보기 애니메이션**:

  - 노드 간 이동이나 관계 변경 시 시각적 피드백 제공
    - **형제 이동**: 동일 부모를 공유할 경우 애니메이션
    - **이웃 이동**: 동일 세대이지만 부모가 다를 경우 애니메이션
    - **세대 이동**: 세대가 바뀌는 경우 애니메이션 적용

- **Pull Animation**:
  - 외부 노드를 추가할 때의 애니메이션으로 시각적 일관성 제공

---

### 4. **기술 스택**

- **프론트엔드**: next.js, TypeScript, D3.js (시각적 데이터 맵핑 및 노드 이동을 위한 라이브러리)
- **백엔드**: django
- **애니메이션**: CSS, GreenSock Animation Platform (GSAP)
- **상태 관리**: Redux or Zustand (노드 상태 및 부모-자식 관계 관리)

---

### 5. **MVP 기능 개발 이후 추가 개선사항**

1. **자동 최적화 기능**: 노드가 많아질 경우 자동으로 간격을 조정하여 스크롤 없이 화면에 표시할 수 있는 기능
2. **멀티 셀렉션 및 이동**: 여러 노드를 동시에 선택하여 부모를 변경하거나 포함 관계를 재정의
3. **커스터마이징**: 노드 색상, 크기, 배경 설정 등 인터페이스 요소의 커스터마이징 기능
4. **데이터 내보내기/가져오기**: JSON 또는 CSV 형식으로 마인드맵 데이터를 내보내거나 가져올 수 있는 기능
5. **실시간 협업**: 여러 사용자가 동시에 마인드맵을 편집할 수 있는 기능 (WebSocket을 활용한 실시간 동기화)
