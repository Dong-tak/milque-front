# PRD: 마인드맵 인터페이스 개발

---

## 1. 프로젝트 개요

이 프로젝트는 사용자가 직관적으로 마인드맵을 생성하고 편집할 수 있는 웹 인터페이스를 개발하는 것을 목표로 합니다. 각 노드는 위젯 형태로 구성되며, 노드 간의 관계와 구조는 특정한 규칙과 간격(Gap)을 기반으로 자동 배치됩니다. 사용자는 노드 생성, 외부 노드 포함, 다양한 애니메이션 효과 등을 통해 풍부한 마인드맵을 구성할 수 있습니다.

---

## 2. 유저 플로우

1. **마인드맵 시작**

   - 사용자가 웹 애플리케이션에 접속하여 새로운 마인드맵을 생성합니다.
   - 중심이 되는 **Root 노드**가 화면에 표시됩니다.

2. **노드 생성**

   - 사용자는 Root 노드나 다른 노드를 선택하여 **형제 노드** 또는 **자식 노드**를 생성할 수 있습니다.
   - 노드를 선택한 후, 컨텍스트 메뉴나 버튼을 통해 원하는 노드 유형을 추가합니다.

3. **외부 노드 포함**

   - 마인드맵 외부에 있는 노드를 마인드맵에 포함시킬 수 있습니다.
   - 특정 노드를 선택한 상태에서 **외부 노드 포함 기능**을 활성화합니다.
   - 외부 노드를 클릭하면 해당 노드가 선택한 노드의 자식으로 마인드맵에 추가됩니다.
   - 전체 마인드맵을 선택한 상태에서 실행하면 외부 노드가 Root 노드의 첫째 자식으로 포함됩니다.

4. **노드 인터랙션**

   - 노드에 마우스를 올리면(hover) **헤드바(headBar)**가 나타나 추가 옵션을 제공합니다.
   - 노드의 상세 정보를 확인하거나 편집할 수 있습니다.

5. **애니메이션 및 미리보기**

   - 노드를 추가하거나 이동할 때, 해당 동작에 대한 **미리보기 애니메이션**이 제공됩니다.
   - **Pull Animation**을 통해 외부 노드를 포함시킬 때 시각적인 효과를 제공합니다.

6. **포지션 고정**
   - Root 노드를 제외한 모든 노드는 자동으로 배치되며, 사용자가 임의로 이동할 수 없습니다.
   - 노드의 위치는 정의된 규칙과 간격에 따라 계산됩니다.

---

## 3. 핵심 기능

### 3.1 노드 관리

- **노드 생성**

  - **형제 노드 추가**: 선택한 노드와 동일한 부모를 가진 노드를 생성합니다.
  - **자식 노드 추가**: 선택한 노드의 자식 노드를 생성합니다.

- **외부 노드 포함**
  - 마인드맵 외부의 노드를 마인드맵에 포함시켜 구조를 확장합니다.

### 3.2 인터랙션 및 UI

- **헤드바 표시**

  - 노드에 마우스를 올리면 헤드바가 나타나며, 노드 편집 및 추가 옵션을 제공합니다.

- **미리보기 애니메이션**

  - 노드를 추가하거나 이동할 때 실시간으로 변화 내용을 시각적으로 보여줍니다.
  - 이동 유형에 따라 다른 애니메이션 효과를 제공합니다:
    - **형제 이동**: 부모와 동일한 노드로 이동할 때.
    - **이웃 이동**: 동일한 세대이지만 부모가 다른 노드로 이동할 때.
    - **세대 이동**: 다른 세대로 노드를 이동할 때.

- **Pull Animation**
  - 외부 노드를 마인드맵에 포함시킬 때 부드러운 애니메이션 효과를 제공합니다.

### 3.3 자동 배치 및 레이아웃

- **포지션 고정**

  - Root 노드를 제외한 모든 노드는 자동으로 계산된 위치에 고정됩니다.
  - 노드 간 간격(Gap)은 정의된 상수에 따라 계산됩니다:
    - **GenGap (48px)**: 부모와 자식 노드 간의 수평 간격.
    - **SibGap (24px)**: 형제 노드 간의 수직 간격.
    - **NavGap (36px)**: 부모가 다른 노드 그룹 간의 수직 간격.

- **Family Box 구조**
  - 노드와 그 자식 노드들을 포함하는 가상의 박스(Family Box)를 사용하여 레이아웃을 계산합니다.
  - 노드의 위치와 크기는 Family Box의 좌표를 기준으로 결정됩니다.

## 4. 기술 스택

- **프론트엔드**

  - **next.js**: 사용자 인터페이스 개발을 위한 라이브러리.
  - **TypeScript**: 정적 타입 체크를 통한 안정적인 코드 작성.
  - **D3.js 또는 Konva.js**: SVG 또는 Canvas를 활용한 시각화 및 노드 배치.
  - **Styled-Components**: 컴포넌트 기반의 스타일링.

- **상태 관리**

  - **Redux** 또는 **Context API**: 애플리케이션 상태 관리.

- **애니메이션**

  - **Framer Motion** 또는 **GSAP**: 부드러운 애니메이션 효과 구현.

- **백엔드 (필요 시)**
  - **Node.js (Express)**: API 서버 구축.
  - **Firebase**: 실시간 데이터베이스 및 인증.
