## 인터페이스

- node : 마인드맵을 구성하는 하나하나의 widget
- Root : 중심 노드
- First : 첫째 자식, 다른 세대와 달리 Left/Right를 구분함
- DBrelate : 관계
  - 요소 관계 : 마인드맵 RelationshipType내에 존재하는 모든 노드와 보드 사이의 관계
  - 포함 관계 : 부모와 하위 자식 노드 사이의 관계
  - 순서 참조 관계 : sibling 사이의
- Box : 특정 노드를 묶은 가상의 박스
  - GenBox : 같은 세대의 SibBox들을 묶는 가상의 박스
  - SibBox : 같은 부모를 공유하는 sibling을 묶는 가상의 박스
- Gap : 인터페이스적으로 노드나 박스 사이의 거리를 나타내는 상수
  - GenGap : GenBox 사이의 horizental space
    - 48
  - SibGap : sibling 사이의 vertical space
    - 24
  - NavGap : 조상을 공유하는 같은 세대 내의 SibBox 사이의 vertical space
    - 36

### PositionFix → 결국 추산하고 싶은 것은 모든 노드의 XY좌표

- Root는 이동할 수 있음 (포지션픽스의 영향을 받지 않음). 나머지 노드는 이동 불가.
- 노드 $N^n_i$는 n번째 세대의 i번째 노드이고, $x^n_i, y^n_i, x^n_i+w^n_i, y^n_i+h^n_i$ 네 점을 범위로 갖습니다.
  - 최상위 노드인
- family box $F^n_i$는 n번째 세대의 i번째 노드를 부모로 삼는 family box이고, $X^n_i, Y^n_i, X^n_i+W^n_i, Y^n_i+H^n_i$ 네 점을 범위로 갖습니다.
  - 최상위
  - 다음 세대가 더이상 없는 경우에는 해당 노드와 같은 사이즈로 설정됩니다.
- i 대신 `m`을 쓰면 같은 세대 중 최댓값이라는 의미를 갖습니다.
  ${w^n_{m}}=\max({w^n_{1}},{w^n_{2}}...{w^n_{i}})$
- z는 갯수를 나타냅니다.
- Gap은 노드나 박스 사이에 존재하는 상수의 빈공간입니다.
  $Gap_{gen}=48, Gap_{sib}=24, Gap_{nav}=36$
  - GenGap : GenBox 사이의 horizental space
  - SibGap : sibling 사이의 vertical space
    - 이제 위에 2개만 씀
- GenBox
  - LeftGenBox
    - Xn = 윗 세대
    - Yn
    - Wn
    - Hn
  - RightGenBox
- node
  - $w^n_i$
    - 초기 : 컨텐츠와 위젯 상태에 따라 결정된 상태
  - $h^n_i$
    - 초기 : 컨텐츠와 위젯 상태에 따라 결정된 상태
  - $x^n_i=X^{n}_i=X^{n-1}_i +w^{n-1}_m$
    - 초기 : 최상위 노드($n=1$)인 루트만 결정된 상태
  - $y^n_i=Y^n_i+\sum^{i-1}_{z=1}({h^n_i+Gap_{sib}})$
    - 초기 : 최상위 노드($n=1$)인 루트만 결정된 상태
- FamilyBox
  - $W^n_i=w^n_m+W^{n+1}_i+Gap_{gen}$
    - 각 세대의 노드 중 최대 너비와 다음 세대의 family box의 너비의 합
    - 즉, 세대별로 familyBox의 width는 모두 같음
  - $H^n_i = \sum_{i=1}^{z}{H^{n+1}_z} + Gap_{sib}\times(z-1)$
    - 다음 세대의 familyBox 높이의 합과 갭의 합
  - $X^n_i=X^{n-1}_i + W^{n-1}_i-W^{n}_i-Gap_{gen}=X^{n-1}_i +w^{n-1}_m$
    -
  - $Y^n_i=Y^{n-1}_i-0.5 H^{n-1}_i+0.5H^n_i$
    - 이전 세대의 (노드 or familyBlock)의 중심 기반, 뭘써도 됨
  - Wn = max(Wn-1) + Nw + Ggen
  - Hn = max(Hn-1, Nh)
